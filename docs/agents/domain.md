# Domain docs

This repository uses a single-context domain layout.

## Before exploring

- Read `CONTEXT.md` at the repository root when it exists.
- Read relevant decisions under `docs/adr/` when they exist.
- Proceed silently when either location does not yet exist; producer workflows create them when terminology or decisions become durable.

## Vocabulary

Use terms exactly as defined in `CONTEXT.md` in issues, plans, tests, and implementation. If a required concept is missing, determine whether it is unnecessary jargon or a genuine domain-language gap before adding it.

## Decisions

If a proposal conflicts with an ADR, identify the conflict explicitly and explain why the decision may be worth reopening rather than silently overriding it.
