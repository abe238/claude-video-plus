## What was completed

### Product and runtime

- Preserved the original `/watch` behavior and automatic compatibility fallback.
- Added opt-in question-aware evidence mode so targeted questions receive relevant chapters,
  transcript spans, numbers, specifications, cue moments, and matching frames instead of the
  entire video timeline.
- Added a recorded evidence manifest explaining what was selected and why.
- Added bounded frame deduplication and dependency-free visual-change scoring.
- Evaluated OpenCV and rejected it from v1 because its prototype was slower and did not improve
  the measured recall or duplicate result.
- Added more resilient, classified video-download attempts, including bounded YouTube recovery
  and optional browser-cookie support without logging cookie values.
- Added better caption-language selection and local VTT/SRT sidecar preference.
- Added local-first transcription routing, including detected YAP and loopback
  OpenAI-compatible `:8082` services, before explicitly authorized cloud transcription.
- Added chunked transcription, bounded retries, resumable receipts, and transcript reuse.
- Added optional local and explicitly authorized remote semantic reranking. The default remains
  dependency-free and sends nothing remotely.
- Added safe evidence state, portable export/verification, bounded storage, cleanup controls,
  and protections against exporting secrets, private absolute paths, or media by accident.
- Kept Fathom and OpenCV outside the v1 runtime instead of adding unproven dependencies.

### Installation and distribution

- Kept the original project's simple Agent Skills installation pattern.
- Made `skills/watch/` a self-contained installable unit whose scripts resolve relative to the
  installed skill.
- Synchronized version `1.0.0` across the skill, Claude plugin, Codex plugin, runtime receipts,
  tests, changelog, README, website, and release links.
- Built and published a stable, non-prerelease `watch.skill` artifact.
- Tested an anonymous public release download and verified its checksum.
- Tested the documented public `npx skills add` command in an isolated temporary home; it found
  and copied the v1.0.0 `watch` skill successfully.
- Updated GitHub Pages to use normal language, show the actual small benchmark numbers, link the
  stable release, and explain limitations without claiming universal superiority.

### Evidence and verification

- Preserved upstream commit `83da59f` as the untouched comparison reference.
- Added deterministic control, release-integrity, bundle, lifecycle, privacy, retrieval,
  semantic, transcription, and portability tests.
- Passed **335 local deterministic tests** in the final verification pass.
- Passed the release validator covering **45 requirements and 35 implementation packets**.
- Passed the hosted release workflow, including the full test suite, compilation, execution-plan
  validation, deterministic bundle construction, and GitHub release creation.
- Published the existing one-video, three-question blind comparison. Evidence mode used an
  estimated **60–79% fewer reader tokens** and tied or won those three questions. Public wording
  explicitly says this is a small initial result, not proof for every video.

## What is still missing

### Optional research for stronger public claims

These are useful if the project later wants to claim that it is broadly better than the original.
They are not required to use or distribute v1.0.0.

1. Finish the multi-video development comparison across varied video types.
2. Have two humans independently annotate the expected evidence and validate the answer grader.
3. Replace the sealed confirmation placeholders with an untouched confirmation set, then run it
   exactly once against a frozen candidate.
4. Publish complete speed, reliability, recall, duplicate-removal, token, and answer-quality
   results, including failures and underpowered categories.
5. Revisit optional mechanisms only when an ablation demonstrates a measurable improvement after
   installation and complete runtime cost are counted.

Until that work exists, keep performance wording limited to the published one-video,
three-question result.

### GitHub fork-network status

GitHub currently reports `parent: null`. GitHub does not provide a normal setting that attaches an
existing repository retroactively to another repository's fork network. Recreating the repository
as a GitHub-native fork could lose or disrupt releases, issues, Pages, and configuration, so that
destructive migration was not performed.

The project is nevertheless unambiguous about its origin through preserved upstream history,
license, authorship, gratitude, repository description, provenance records, and the `fork` topic.
If native fork-network membership becomes essential, contact GitHub Support about attaching the
repository without recreating it before attempting any migration.

## Publication decision

`v1.0.0` is published, public, installable, and ready for normal use. There is no remaining release
blocker. The only open work is broader comparative research and the optional GitHub fork-network
administrative question described above.
